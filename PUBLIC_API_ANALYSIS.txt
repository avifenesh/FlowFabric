# FlowFabric Glide-Core Public API Surface

## Re-Exported Types (from lib.rs)

### Direct Re-exports:
- `pub use client::ConnectionRequest` - Connection configuration
- `pub use client::StandaloneClient` - Standalone client type (from client/mod.rs:24)
- `pub use types::*` - All types from client::types (from client/mod.rs:32)
- `pub use reconnecting_connection::IAMTokenHandle` - IAM token management (from client/mod.rs:36)

## API Stability Assessment

### Stable (Unlikely to Change):
- `ConnectionRequest` - Core config type
- `StandaloneClient` - Stable interface
- `RequestType` - Command enum (only grows)
- `TlsMode`, `ReadFrom` - Configuration enums
- `ConnectionError` - Error propagation

### Semi-Stable (May Evolve):
- `Client` - Main client interface (could add methods)
- `AuthenticationInfo` - Auth config (may add new auth types)
- `CompressionConfig` - Compression (may add backends)

### Internal/Unstable:
- `ClientWrapper` - Internal enum
- `LazyClient` - Implementation detail
- `PubSubCommandApplier` - Test-facing trait
- `GlideClientForTests` - Test infrastructure

## Hidden/Internal Implementation Details

These are public but not primary entry points:
- `get_or_init_runtime()` - Runtime management (usually implicit)
- `get_valkey_connection_info()` - Parsing helper (low-level)
- `cluster_scan_container` - Implementation detail for SCAN
- `otel_db_semantics` - OTel implementation

## API Gaps

Potential missing functionality:
1. No direct connection pooling control (uses internal pooling)
2. No metrics collection interface (OTel only)
3. Limited command-level tracing granularity
4. No explicit connection lifecycle hooks
