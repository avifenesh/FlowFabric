## Consumer-ergonomics notes (not bugs, but friction)

Imports required slightly more precision than a naive consumer would guess:

1. `ScannerFilter` is NOT re-exported at `ff_core::ScannerFilter` — must use `ff_core::backend::ScannerFilter`.
2. `BackendConfig` is NOT re-exported at `ff_backend_valkey::BackendConfig` — must use `ff_core::backend::BackendConfig`. (The `pub use` at `ff_backend_valkey/src/lib.rs:32` shadows with a private name.)
3. `Namespace` is NOT in `ff_core::backend` — must use `ff_core::types::Namespace`.
4. `ScannerFilter::with_namespace` takes a typed `Namespace` (no `impl Into<Namespace>`), so `&str` → three extra characters of ceremony per call.

None of these are regressions vs. the Worker X v0.3.2 findings the 0.3.x line was supposed to fix — they're pre-existing discoverability papercuts. The 0.3.2 report's core regressions (if any around these four APIs) land clean on 0.3.4.
