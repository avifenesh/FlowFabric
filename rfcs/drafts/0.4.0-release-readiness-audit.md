## Breaking change inventory

Per crate, for everything between `v0.3.4` and current `main`:

### `ff-core`

1. `BackendError` + `BackendErrorKind` added (new, non-exhaustive).
2. `backend::Frame` — added fields `frame_type: String`,
   `correlation_id: Option<String>`; still `#[non_exhaustive]`, but
   existing `pub` field-set patterns at callers rely on the new
   fields for correct semantics.
3. `EngineBackend::append_frame` return type `()` →
   `AppendFrameOutcome` (moved from `ff_sdk::task`).
4. `EngineBackend::create_waitpoint` — new trait method (+
   `PendingWaitpoint` type).
5. `EngineBackend::report_usage` return type `AdmissionDecision` →
   `ReportUsageResult` (non-exhaustive).
6. `AdmissionDecision` removed.
7. `UsageDimensions::dedup_key: Option<String>` added (non-exhaustive
   struct so additive, but wire-semantic change).
8. `BackendTimeouts.keepalive` field removed.
9. `BackendRetry` fields reshaped: `max_attempts`/`base_backoff` →
   `exponent_base`/`factor`/`number_of_retries`/`jitter_percent`.

### `ff-sdk`

10. `SdkError::Valkey(ferriskey::Error)` → `SdkError::Backend(BackendError)`.
11. `SdkError::ValkeyContext { source: ferriskey::Error, .. }` →
    `SdkError::BackendContext { source: BackendError, .. }`.
12. `SdkError::valkey_kind` renamed to `backend_kind` and return type
    changes from `Option<ferriskey::ErrorKind>` to
    `Option<BackendErrorKind>`.
13. `ClaimedTask::append_frame` return `()` → `AppendFrameOutcome`
    (shim `pub use` keeps the path; type is still re-exported).

### `ff-server`

14. `ServerError::Valkey` / `ValkeyContext` → `Backend(BackendError)` /
    `BackendContext`.
15. `ServerError::valkey_kind` → `backend_kind`.
16. HTTP `ErrorBody.kind` wire-string values change (`"io_error"` →
    `"transport"`, cluster-redirect kinds → `"cluster"`, etc.) —
    downstream-consumer-visible.

### `ff-backend-valkey`

17. `append_frame` / `create_waitpoint` / `report_usage` bodies move
    from `Unavailable` stubs to real Lua-backed impls. External impls:
    none in-tree; behavioral change visible to anyone who was exercising
    the stubs.
18. `BackendRetry` wired to `ClientBuilder::retry_strategy` (first-time
    wiring; not a signature change but changes observed connection
    behavior when callers set these fields).
19. `BackendTimeouts.request` wired to `ClientBuilder::request_timeout`.

### Pending (#146 + later tranches)

20. `WorkerConfig::new` removed (T1).
21. `WorkerConfig` field split: `host`/`port`/`tls`/`cluster` move into
    nested `backend: BackendConfig` (T1).
22. `ff_backend_valkey::build_client` becomes `pub` (T1, additive).
23. Snapshot decoder `EngineError::Validation{Corruption}` reshape (T3).
24. Seal `FlowFabricWorker::client()` + relocate stream free-fns (T4,
    closes #87).

Count: **19 breaking items merged**, **5 more expected** via the Stage
1c bundle for a total of ~24.
